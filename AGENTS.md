# redbox-hook--researchdatabox-sails-hook-redbox-supportagreements Agent Guide

This hook follows the standard ReDBox hook archetype generated by `@researchdatabox/redbox-dev-tools init`.

Keep these rules in place unless the user explicitly asks to change the hook model.

## Dependency Contract

- `@researchdatabox/redbox-core` is the runtime compatibility anchor.
- `@researchdatabox/redbox-dev-tools` is the shared authoring and test toolchain.
- Direct hook `dependencies` are reserved for hook-owned runtime packages only.

Do not directly add shared runtime or toolchain packages such as:

- `axios`
- `rxjs`
- `lodash`
- `typescript`
- `ts-node`
- `mocha`
- `chai`
- `@researchdatabox/sails-ng-common`

## Compile And Test

Use the existing scripts in `package.json`.

Do not replace the custom `run-hook-tsc` or `run-hook-mocha` entrypoints with plain `tsc` or `mocha`.

## Hook Runtime

This hook copies `views/` into the portal during `ConfigService.mergeHookConfig()`. Keep view paths compatible with core view lookup.

## Docker Runtime

The docker compose development stack builds a ReDBox portal image with this hook installed, then bind-mounts the hook directory into the portal's `node_modules` path for local development.

The compose file intentionally sets `NODE_PATH` so the mounted hook can resolve host-provided runtime dependencies from the ReDBox portal dependency tree. Do not remove that unless the runtime install model changes deliberately.
